Index Initializers

new TestObj { [index] = value, [index2] = value2 } 

//C# 6
var numbers = new Dictionary<int, string> { 
    [7] = "seven", 
    [13] = "thirteen" 
};
//C# 5
var numbers = new Dictionary<int, string> { 
    {7, "seven"}
    {13, "thirteen"}
};

Index Initializers - Why?

  • In C# 5, class must have Add method and implemented IEnumerable to initialize it using {index, value} syntax.
  • In C# 6, using index initializer syntax, [index] = value, class only needs an indexer method, no base class or interface required.

Exception Filters

Add when (expression) to end of a catch clause. Exception is caught only if expression is true.

//C# 6
try {
  //...
} catch (ArgumentException e) when (e.ParamName == "shoppingCart") {
  abortSale();
}
//C# 5
try {
  //...
} catch (ArgumentException e)
  if (e.ParamName == "shoppingCart") 
	abortSale();
  throw;
}

Exception Filters - Why?

  • In C# 5, for equivalent behavior, you had to check your condition and if not true, you had to re-throw the exception. This pollutes the exception stack trace, it will now immediately reference the line of your throw;
    • statement.
  • In C# 6, if the catch expression is false, nothing about the exception stack trace is affected.

Exception Filters - Why? (Continued)

  • In C# 6, the catch expression can have side-effects and can always return false. This allows exception logging without affecting the exception stack trace.
    • //C# 6
try {
  //...
} catch (Exception e) when (LogException(e)) {}

Async/Await in catch/finally

  • In C# 5, await could only be used in body of try, a compile error would result if await was used in body of catch or finally.
  • In C# 6, await can now be used in try, catch and finally blocks.
  • Why? Makes it much easier to support asynchronous error handling and logging.

using static directive

  • In C# 5, using directive only imports a namespace.
  • In C# 6, the new using static directive imports static members of specified type. This includes enumeration values.
  • Why? The same benefit of using directive to shorten commonly typed namespaces can now be applied to static members of types with using static directive.

using static directive (Example)

//C# 6
using static System.Console;
using static System.ConsoleColor; //enumeration
using static System.Math;
// ...
  void Calculate(double value) {
    ForegroundColor = Blue;
    WriteLine("Result is " + Sqrt(value));
  }

//C# 5
using System;
// ...
  void Calculate(double value) {
    Console.ForegroundColor = ConsoleColor.Blue;
    Console.WriteLine("Result is " + Math.Sqrt(value));
  }

Null Conditional Access, ?.

  • ?. can be used instead of . and returns nullable version of right-hand side if left-hand side is null.
  • ?[] can be used instead of [] and returns nullable version of right-hand side if left-hand side is null.
  • Prevents NullPointerExceptions from being thrown.
//C# 6
void TestMethod(Customer person) {
  string name = person?.FirstName;
  int? age = person?.Age; //Age is int
  bool? inUs = person?.Address?.IsInUS(); //IsInUs() method returns bool
  string recentOrder = person?.Orders?[0]?.OrderId;
  //throws IndexOutOfRangeException if Orders is non-null but empty
}

Null Conditional Access, ?. - Why

  • In C# 5, you have to write if statement or ternary (?:) to check for nulls. ?. and ?[] replace those in C# 6.
  • Code is now easier to read, also works well with null coalescing operator (??).
  • Fewer exceptions, with less code, who doesn't like that?
    • //C# 6
var zip = person?.Address?.ZipCode ?? "Unknown Zip";

    //C# 5
var zip = "Unknown Zip";
if (person != null &&
    person.Address != null &&
    person.Address.ZipCode != null) {
  zip = person.Address.ZipCode;
}

nameof operator

Returns a string, which is the unqualified name of a variable, type or member.

//C# 6
void TestMethod(Customer person) {
  nameof(person); // returns "person" - variable
  nameof(person.Address.ZipCode); // returns "ZipCode" - member
  nameof(System.DateTime); //returns "DateTime" - type
  nameof(System.Collections.Generic.List<int>); //returns "List" - type
}

nameof operator - Why?

  • In C# 5, you would have to manually type the string, which means it can become out-of-date. With nameof operator, a compile error occurs if the variable, type or member is renamed.
  • Useful with ArgumentNullException and ArgumentException.
  • Useful with XAML, WPF implementations of INotifyPropertyChanged interface.

String Interpolation, $""

In-line template strings, use $"" instead of "" where {expression} is replaced with the ToString representation of the expression.

//C# 6
void Format(string name, DateTime startDate, double salary) {
  var tag = $"{name}, hired {startDate:D}";
  var wage = $"Paid: {salary:c}";
  var desc = $"Is {(startDate.Year < 2010 ? "pre" : "post")} reorg";
}

//C# 5
void Format(string name, DateTime startDate, double salary) {
  var tag = String.Format("{0}, hired {1:D}", name, startDate);
  var wage = String.Format("Paid: {0:c}", salary);
  var desc = String.Format("Is {0} reorg",
    (startDate.Year < 2010 ? "pre" : "post"));
}

String Interpolation, $"" - Why?

  • More concise that String.Format, but exactly equivalent. Supports format specifiers and alignment syntax.
  • No need to worry about mis-matching {0} and arguments to String.Format.
  • Still have intellisense for expressions and compile-time errors if variables are renamed.
  • Can be assigned to System.IFormattable and System.FormattableString if you want to use culture specific rendering of the template (e.g. globalization).
